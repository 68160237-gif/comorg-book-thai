# วิวัฒนาการของหน่วยความจำหลัก (Main Memory)
ยุคก่อน (Core Memory): คอมพิวเตอร์สมัยก่อนใช้หน่วยความจำแบบ Magnetic Core ซึ่งทำจากห่วงเฟอร์โรแมกเนติกรูปโดนัท เรียงต่อกันเป็นแถวลำดับ (Array)
เกร็ดน่ารู้: แม้ปัจจุบันเทคโนโลยีนี้จะเลิกใช้ไปแล้ว แต่คำว่า "Core" ยังคงถูกใช้เรียกแทนหน่วยความจำหลักมาจนถึงทุกวันนี้
จุดเปลี่ยน: การเกิดขึ้นของเทคโนโลยี Microelectronics (ไมโครอิเล็กทรอนิกส์) เข้ามาแทนที่ด้วยข้อดีที่เหนือกว่าในหลายด้าน
ยุคปัจจุบัน (Semiconductor Memory): หน่วยความจำในปัจจุบันเกือบทั้งหมดเปลี่ยนมาใช้ Semiconductor Chips (ชิปสารกึ่งตัวนำ) เป็นมาตรฐานสากลไปแล้ว
**โครงสร้างพื้นฐาน (Organization)**
หน่วยย่อยที่สุด: Memory Cell (เซลล์หน่วยความจำ)
ไม่ว่าเทคโนโลยีภายในจะต่างกันแค่ไหน แต่ Memory Cell ทุกประเภทจะมีคุณสมบัติร่วมกัน 3 ประการ คือ:
1.มี 2 สถานะที่คงที่: เพื่อใช้แทนค่าเลขฐานสองคือ 0 และ 1
2.เขียนข้อมูลได้: สามารถเปลี่ยนสถานะข้างต้นได้อย่างน้อย 1 ครั้ง
3.อ่านข้อมูลได้: สามารถตรวจสอบหรือรับรู้สถานะปัจจุบันของเซลล์นั้นได้
การทำงานของ Memory Cell (3 ขั้วสัญญาณหลัก)
การทำงานของเซลล์หนึ่งๆ จะอาศัยขั้วสัญญาณ (Terminals) 3 ส่วนประกอบกัน:
Select (ขั้วเลือก): ทำหน้าที่ระบุว่าจะเลือกเซลล์นี้เพื่อทำการอ่านหรือเขียนข้อมูล
Control (ขั้วควบคุม): เป็นตัวสั่งการว่าให้ทำหน้าที่ "อ่าน (Read)" หรือ "เขียน (Write)"
Data Terminal (ขั้วข้อมูล): * ขาเขียน: รับสัญญาณไฟฟ้าเข้ามาเพื่อกำหนดค่า 0 หรือ 1
**สรุปเปรียบเทียบ DRAM และ SRAM**
ลักษณะที่เหมือนกัน:
Volatile: ข้อมูลจะสูญหายทันทีหากไม่มีกระแสไฟฟ้าเลี้ยง (ต้องมีไฟเลี้ยงตลอดเวลา)
Random Access: สามารถเข้าถึงข้อมูลในทุกตำแหน่งได้โดยตรงผ่านสัญญาณไฟฟ้า
ข้อแตกต่างที่สำคัญ:
1. DRAM (Dynamic RAM)
เทคโนโลยี: เก็บข้อมูลในรูปแบบประจุไฟฟ้าใน Capacitor (ตัวเก็บประจุ)
การรักษาข้อมูล: ต้องมีการ Refresh (อัดประจุ) ข้อมูลตลอดเวลา เพราะประจุไฟฟ้ามีการรั่วไหล
ความเร็ว: ทำงานได้ช้ากว่า SRAM
ความหนาแน่น: เซลล์มีขนาดเล็กมาก ทำให้บรรจุข้อมูลได้หนาแน่นและมีปริมาณมาก
ราคา: ราคาต่อความจุถูกกว่า
การใช้งาน: นิยมใช้เป็น หน่วยความจำหลัก (Main Memory) ของคอมพิวเตอร์
2. SRAM (Static RAM)
เทคโนโลยี: เก็บข้อมูลโดยใช้ชุดทรานซิสเตอร์แบบ Flip-flop (Digital Logic)
การรักษาข้อมูล: ไม่ต้องทำการ Refresh ข้อมูลจะคงอยู่ตลอดตราบเท่าที่มีไฟเลี้ยง
ความเร็ว: ทำงานได้เร็วมาก
ความหนาแน่น: เซลล์มีขนาดใหญ่กว่า (ใช้ทรานซิสเตอร์ 4-6 ตัวต่อบิต) ทำให้ความหนาแน่นน้อย
ราคา: ราคาต่อความจุแพงกว่ามาก
การใช้งาน: นิยมใช้เป็น Cache Memory (ทั้งในและนอก CPU)
**สรุปประเภทของหน่วยความจำ ROM**
คุณสมบัติพื้นฐาน:
Nonvolatile: ข้อมูลไม่หายเมื่อไม่มีไฟเลี้ยง
Read-Only: เน้นการอ่านข้อมูลเป็นหลัก (บางชนิดเขียนทับได้ยากหรือมีเงื่อนไข)
การใช้งาน: ใช้เก็บ Micro-programming, Library subroutines, ระบบปฏิบัติการส่วนต้น (System programs) และตารางฟังก์ชันต่างๆ
รายละเอียดเพิ่มเติมที่สำคัญ:
ROM: หากผิดพลาดเพียง 1 บิตในขั้นตอนผลิต ต้องทิ้งทั้งล็อต (Fixed cost สูง)
EPROM: การลบด้วยแสง UV ใช้เวลานาน (ประมาณ 20 นาที) และต้องทำทั้งชิป
EEPROM: แก้ไขข้อมูลได้ง่ายโดยไม่ต้องถอดชิปออก แต่วงจรซับซ้อนทำให้จุข้อมูลได้น้อยต่อหนึ่งชิป
Flash Memory: เป็นเทคโนโลยีที่อยู่ตรงกลางระหว่าง EPROM (ในแง่ความหนาแน่น/ใช้ทรานซิสเตอร์ตัวเดียวต่อบิต) และ EEPROM (ในแง่การลบด้วยไฟฟ้า) แต่เน้นลบข้อมูลเป็น "บล็อก" เพื่อความรวดเร็ว
**สรุปโครงสร้างและตรรกะของชิปหน่วยความจำ (Chip Logic)**
1. การจัดระเบียบข้อมูล (Memory Organization)
ชิปแต่ละตัวประกอบด้วยอาเรย์ของ Memory Cells
มีการออกแบบตั้งแต่แบบ 1-bit-per-chip (อ่าน/เขียนทีละ 1 บิต) ไปจนถึงแบบที่อ่าน/เขียนได้หลายบิตพร้อมกัน (เช่น 4 หรือ 16 บิต)
หากชิปตัวเดียวอ่าน/เขียนได้ไม่ครบจำนวนบิตของ Word (เช่น Word ละ 64 บิต แต่ชิปอ่านได้ทีละ 4 บิต) จะต้องใช้ชิปหลายตัวทำงานร่วมกันผ่าน Memory Controller
2. การระบุตำแหน่ง (Addressing & Pin Saving)
Multiplexed Addressing: เพื่อลดจำนวนขา (Pins) บนชิป จึงใช้วิธีส่งที่อยู่ (Address) แบ่งเป็น 2 รอบผ่านสายชุดเดิม:
Row Address: ระบุแถว โดยมีสัญญาณ RAS (Row Address Select) ควบคุมจังหวะ
Column Address: ระบุคอลัมน์ โดยมีสัญญาณ CAS (Column Address Select) ควบคุมจังหวะ
ผลลัพธ์: การเพิ่มขา Address เพียง 1 ขา จะทำให้ความจุชิปเพิ่มขึ้นถึง 4 เท่า (เพราะเพิ่มทั้งจำนวนแถวและคอลัมน์อย่างละ 2 เท่า)
3. ส่วนประกอบสำคัญบนชิป
Row/Column Decoders: แปลงรหัส Address (เช่น 11 เส้น) ให้กลายเป็นการเลือกแถวหรือคอลัมน์ที่ต้องการ (เช่น 2,048 เส้น)
Sense Amplifier: ทำหน้าที่ขยายสัญญาณไฟฟ้าขนาดเล็กจากเซลล์เพื่ออ่านค่าว่าเป็น 0 หรือ 1
Data Buffer: พักข้อมูลก่อนส่งออกหรือเขียนเข้า
Control Pins: * WE (Write Enable): ควบคุมการเขียน
OE (Output Enable): ควบคุมการอ่าน
Vss / Vcc: สายดินและแหล่งจ่ายไฟ
4. กลไกการรีเฟรช (Refresh Circuitry)
เนื่องจากเป็น DRAM ประจุจะรั่วไหล จึงต้องมีวงจร Refresh ภายใน
วิธีการ: ใช้ Refresh Counter ไล่ไปทีละแถว (Row) โดยการอ่านข้อมูลออกมาแล้วเขียนกลับเข้าไปใหม่ในตำแหน่งเดิมเพื่อเติมประจุให้เต็ม
**สรุปการแพ็กเกจชิปหน่วยความจำ (Chip Packaging)**
ชิปหน่วยความจำจะถูกบรรจุลงในแพ็กเกจที่มี Pins (ขา) สำหรับเชื่อมต่อกับภายนอก โดยมีตัวอย่างที่สำคัญ 2 รูปแบบดังนี้:
1. แพ็กเกจ EPROM (ตัวอย่างขนาด 8-Mbit, จัดระเบียบแบบ 1M x 8)
เป็นชิปที่อ่านข้อมูลออกมาทีละ 1 Word (8 บิต) ประกอบด้วยขาที่สำคัญคือ:
Address Pins (A0–A19): ใช้ 20 ขา เพื่อระบุตำแหน่ง 1 ล้านตำแหน่ง ($2^{20} = 1M$)
Data Pins (D0–D7): ใช้ 8 ขา สำหรับส่งข้อมูลออก (Read)
CE (Chip Enable): ใช้ระบุว่า Address ที่ส่งมานั้นเป็นของชิปตัวนี้หรือไม่ (ใช้เมื่อมีชิปหลายตัวต่อบนบัสเดียวกัน)
Vpp (Program Voltage): แรงดันไฟฟ้าพิเศษที่ใช้เฉพาะตอนเขียนข้อมูล (Programming)
Vcc / Vss: ขาไฟเลี้ยงและสายดิน
2. แพ็กเกจ DRAM (ตัวอย่างขนาด 16-Mbit, จัดระเบียบแบบ 4M x 4)
มีความแตกต่างจาก ROM/EPROM ในหลายจุดเพื่อความประหยัดและความยืดหยุ่น:
Address Pins: ใช้เพียง 11 ขา (แทนที่จะเป็น 22 ขา) เพราะใช้วิธี Multiplexed Address แบ่งส่งรอบแถวและคอลัมน์ ($2^{11} \times 2^{11} = 2^{22} = 4M$)
Data Pins: เป็นแบบ Input/Output (I/O) ในขาเดียวกัน เพราะ RAM ต้องทั้งอ่านและเขียนได้
Control Pins: * RAS / CAS: ควบคุมจังหวะการรับ Address แถวและคอลัมน์
WE / OE: ระบุว่าจะทำการเขียน (Write Enable) หรืออ่าน (Output Enable)
NC (No Connect): ขาที่ไม่ได้เชื่อมต่อกับวงจรภายใน เพื่อให้แพ็กเกจมีจำนวนขาเป็นเลขคู่ตามมาตรฐาน
**สรุปการจัดระเบียบโมดูลหน่วยความจำ (Module Organization)**
เมื่อชิปหน่วยความจำหนึ่งตัวมีความจุหรือจำนวนบิตไม่เพียงพอต่อความต้องการของระบบ เราจะนำชิปหลายตัวมาต่อกันเป็น Module โดยมีหลักการดังนี้:
1. การขยายความกว้างของข้อมูล (เพิ่มจำนวน Bit ต่อ Word)
หากชิปหนึ่งตัวเก็บข้อมูลได้เพียง 1 บิตต่อตำแหน่ง (1-bit-per-chip) แต่ระบบต้องการ Word ขนาด 8 บิต
วิธีการ: นำชิป 8 ตัวมาวางขนานกัน โดยให้ทุกตัวรับ Address ชุดเดียวกัน
ผลลัพธ์: เมื่อสั่งอ่านข้อมูล ชิปแต่ละตัวจะส่งออกมาตัวละ 1 บิต รวมกันเป็น 1 Word (8 บิต) ครบพอดี
2. การขยายความจุของหน่วยความจำ (เพิ่มจำนวน Word)
หากต้องการหน่วยความจำที่มีจำนวน Word มากกว่าความจุของชิปตัวเดียว (เช่น ต้องการ 1M Word แต่ชิปมีความจุแค่ 256K)
วิธีการ: จัดกลุ่มชิปเป็นชุดๆ (Columns/Banks) และใช้ Group Select Logic
ตัวอย่าง (จากเนื้อหา): ต้องการสร้างหน่วยความจำขนาด 1M x 8 บิต จากชิปขนาด 256K บิต
**สรุปหน่วยความจำแบบอินเตอร์ลีฟ (Interleaved Memory)**
หลักการทำงาน:
Memory Bank: เป็นการรวมกลุ่มของชิป DRAM เข้าด้วยกันเป็น "ธนาคารหน่วยความจำ" (Bank)
อิสระต่อกัน: แต่ละ Bank สามารถจัดการคำขออ่าน (Read) หรือเขียน (Write) ข้อมูลได้ด้วยตัวเองอย่างเป็นอิสระ
-ข้อดีและประสิทธิภาพ:
ทำงานพร้อมกัน (Simultaneous Access): หากระบบมีจำนวน K Banks จะสามารถรองรับคำขอได้พร้อมกันถึง K คำขอ
เพิ่มความเร็ว: ช่วยเพิ่มอัตราการรับส่งข้อมูล (Data Transfer Rate) ได้สูงสุดถึง K เท่า
การเก็บข้อมูลแบบสลับ: โดยปกติจะเก็บข้อมูลคำ (Words) ที่อยู่ติดกันไว้ใน คนละ Bank กัน (Consecutive words in different banks)
-เปรียบเทียบเชิงประสิทธิภาพ
แบบธรรมดา: ต้องรออ่านข้อมูลทีละคำจาก Bank เดียวจนครบ
แบบ Interleaved: กระจายการอ่านไปยังหลาย Bank พร้อมกัน (Parallelism) ลดคอขวดของความเร็วหน่วยความจำ
