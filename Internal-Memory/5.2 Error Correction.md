# **การแก้ไขข้อผิดพลาด (Error Correction)**

## 1. ประเภทของข้อผิดพลาด (Types of Memory Failures)
เราสามารถแบ่งปัญหาที่เกิดกับแรมหรือหน่วยความจำได้เป็น 2 ลักษณะ:


ความล้มเหลวแบบถาวร (Hard Failure):

        -อาการ: ฮาร์ดแวร์เสียหายโดยตรง บิตข้อมูลค้าง (Stuck at 0 or 1)

        -ผลกระทบ: ตำแหน่งนั้นจะใช้งานไม่ได้ตลอดไปจนกว่าจะเปลี่ยนอุปกรณ์

   
ข้อผิดพลาดแบบชั่วคราว (Soft Error):


        -อาการ: ข้อมูลเพี้ยนแบบสุ่ม (เช่น 0 กลายเป็น 1) โดยที่ตัวชิปไม่ได้เสีย


        -สาเหตุ: สัญญาณรบกวนทางไฟฟ้า หรืออนุภาคจากรังสี (Cosmic rays)


## 2. หลักการทำงานของรหัสแก้ไขข้อผิดพลาด (ECC)

   
ระบบจะสร้าง บิตตรวจสอบ (Check Bits) เพิ่มเข้าไปในข้อมูลหลัก เพื่อใช้ตรวจสอบและซ่อมแซมตัวเอง


        ขั้นตอนการเขียน (Write): นำข้อมูล M บิต มาคำนวณผ่านฟังก์ชัน f เพื่อสร้าง Check Bits จำนวน K บิต แล้วบันทึกรวมกันเป็น M+K


        บิตขั้นตอนการอ่าน (Read): นำข้อมูล M บิตที่อ่านได้มาคำนวณใหม่ แล้วนำไป เปรียบเทียบ กับ Check Bits เดิมที่เก็บไว้

## 3. ผลลัพธ์จากการเปรียบเทียบ (Comparison Results)

 
เมื่อตรวจสอบแล้ว ระบบจะตัดสินใจดำเนินการใน 3 รูปแบบ:


กรณี        | สถานะ                    | การดำเนินการ
|---|---|---|
ปกติ         |  No Error Detected       | ข้อมูลถูกต้อง ส่งออกไปใช้งานทันที
แก้ไขได้      | Single-bit Error         |พบที่ผิด 1 ตำแหน่ง ระบบ ซ่อมแซมให้โดยอัตโนมัติ
แก้ไขไม่ได้    |Uncorrectable Error       | ผิดพลาดหลายบิตเกินกำลังรหัส ระบบจะ แจ้งเตือน (Error Report)


## 4. มาตรฐาน SEC-DED (Hamming Code)


รหัสแฮมมิง (Hamming Code) เป็นกลไกที่นิยมที่สุดในระบบ ECC Memory โดยมีมาตรฐานที่เรียกว่า SEC-DED:


        SEC (Single-Error-Correcting): แก้ไขได้ทันทีถ้าผิด 1 บิต


        DED (Double-Error-Detecting): ตรวจเจอว่าผิดถ้าพลาดพร้อมกัน 2 บิต (แต่จะแก้ไม่ได้ แค่แจ้งให้ทราบเพื่อป้องกันข้อมูลเสียหายบานปลาย)
