# 5.6 หน่วยความจำภายใน (Internal Memory)
สรุปเนื้อหาสำคัญเกี่ยวกับประเภทของหน่วยความจำเซมิคอนดักเตอร์, การทำงานของ DRAM/SRAM, หน่วยความจำแบบอ่านอย่างเดียว (ROM) และกลไกการตรวจจับและแก้ไขข้อผิดพลาด (ECC)
## 1. คำศัพท์สำคัญ (Key Terms)
### หน่วยความจำภายในสามารถแบ่งกลุ่มใหญ่ๆ ได้ดังนี้:
    -RAM (Random Access Memory): หน่วยความจำแบบสุ่ม เข้าถึงข้อมูลได้เร็ว แบ่งเป็น DRAM (ต้องรีเฟรชข้อมูลตลอดเวลา) และ SRAM (เก็บข้อมูลได้ตราบที่มีไฟเลี้ยง)
 
    -ROM (Read-Only Memory): หน่วยความจำที่เน้นการอ่าน ข้อมูลไม่หายเมื่อไม่มีไฟ แบ่งเป็น PROM, EPROM, EEPROM และ Flash Memory (NAND/NOR)
 
    -Error Correction: กลไกการจัดการความผิดพลาด เช่น Hamming Code, SEC-DED (Single Error Correcting, Double Error Detecting) และ Parity bit
                   
    -Emerging Technologies: เทคโนโลยีใหม่ๆ เช่น MRAM (Magnetic RAM) และ ReRAM (Resistive RAM)
## 2. การเปรียบเทียบ SRAM และ DRAM
| คุณสมบัติ | DRAM (Dynamic RAM) | SRAM (Static RAM) |
| :--- | :--- | :--- |
| **โครงสร้าง** | ใช้ตัวเก็บประจุ (Capacitor) | ใช้ Flip-flop (Transistors) |
| **ความหนาแน่น** | สูง (ขนาดเล็กกว่า) | ต่ำ (ขนาดใหญ่กว่า) |
| **ความเร็ว** | ช้ากว่า | เร็วมาก |
| **ราคา** | ถูกกว่า | แพงกว่า |
| **การใช้งาน** | หน่วยความจำหลัก (Main Memory) | หน่วยความจำแคช (Cache) |
## 3. ประเภทของหน่วยความจำแบบ Non-volatile (ROM & Flash)
    -ROM: เขียนข้อมูลมาจากโรงงาน แก้ไขไม่ได้

    -PROM: เขียนได้ครั้งเดียวโดยผู้ใช้

    -EPROM: ลบข้อมูลได้ด้วยแสง UV และเขียนใหม่ได้

    -EEPROM: ลบและเขียนใหม่ได้ด้วยสัญญาณไฟฟ้า (Byte-level)
    
    -Flash Memory: ลบและเขียนได้เร็วในระดับ Block แบ่งเป็น:
    
    -NAND: ความหนาแน่นสูง ราคาถูก เหมาะสำหรับเก็บไฟล์ (SSD, USB)
    
    -NOR: เข้าถึงข้อมูลแบบสุ่มได้เร็ว เหมาะสำหรับเก็บ Code (Firmware)
## 4. การตรวจจับและแก้ไขข้อผิดพลาด (Error Correction)
### เพื่อป้องกันปัญหา Soft Error (เกิดจากสัญญาณรบกวน) และ Hard Failure (อุปกรณ์พัง) ระบบจึงใช้ Hamming Code ในการจัดการ:
    Hamming Code: การเพิ่ม Check bits เข้าไปในข้อมูลเพื่อระบุตำแหน่งของบิตที่ผิดพลาด
    
    Syndrome: ผลลัพธ์จากการคำนวณ Check bits ตอนอ่านข้อมูล หากเป็น 0 แสดงว่าไม่มีข้อผิดพลาด
### สูตรการคำนวณจำนวน Check bits ($K$):
เพื่อให้สามารถแก้ไขข้อผิดพลาด 1 บิตในข้อมูลจำนวน $M$ บิตได้ ต้องเป็นไปตามเงื่อนไข:
# $$2^K - 1 \ge M + K$$
## 5. การวิเคราะห์ประสิทธิภาพและความน่าเชื่อถือ (Problems & Analysis)
จากโจทย์ในแบบฝึกหัด มีประเด็นที่น่าสนใจดังนี้:

    DRAM Refresh: DRAM ต้องมีการรีเฟรชข้อมูลเป็นระยะ (เช่น ทุกๆ 4ms หรือ 64ms) ซึ่งจะกินเวลาการทำงานของ Memory Cycle ไปบางส่วน
        
    DRAM Timing: การเข้าถึงข้อมูลต้องผ่านขั้นตอนการส่ง Row Address (RAS) และ Column Address (CAS) ทำให้มีค่าหน่วงเวลา (Latency)
        
    Reliability (FIT & MTBF): * FIT (Failure in Time): อัตราการเสียต่อพันล้านชั่วโมงอุปกรณ์ (10^9 hours)
        
    MTBF (Mean Time Between Failures): เวลาเฉลี่ยระหว่างการเกิดความเสียหาย  MTBF = 1/Failure
## 6. สรุปฟังก์ชันการทำงานของชิปหน่วยความจำ (SRAM 7489)
### ชิปหน่วยความจำขนาดเล็ก (เช่น 16 x 4 SRAM) มีโหมดการทำงานหลัก 4 โหมดตามสถานะของขา $\overline{CS}$ (Chip Select) และ $R/\overline{W}$ (Read/Write):
    1.Write: เก็บข้อมูลลงใน Address ที่ระบุ
    
    2.Read: อ่านข้อมูลจาก Address ออกมา
    
    3.Inhibit Writing: ห้ามเขียนข้อมูล
    
    4.Store/Disable: สถานะ Standby ข้อมูลไม่ถูกส่งออก
